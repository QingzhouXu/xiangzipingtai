# AI智能客服对话系统 — 项目全面介绍

## 一、项目概述

这是一个**多商户AI智能客服对话平台**，定位为"比赛演示系统"。系统以Flask为Web框架，集成本地/云端大语言模型(LLM)，为不同商家提供可定制的AI客服机器人，并配套完整的商户管理和平台管理后台。

- **项目名称**：AI智能客服对话系统 (ai-customer-service)
- **运行方式**：`python app.py` 一键启动
- **技术栈**：Python 3.10 + Flask + scikit-learn + PyTorch + Transformers + Ollama/DashScope/OpenAI

## 三、核心模块详解

### 1. LLM多后端客户端 (`src/llm/qwen_client.py`)

**亮点：抽象基类 + 多后端统一接口 + 运行时热切换**

- 定义了 `BaseLLMClient` 抽象基类，统一 `chat()` / `stream_chat()` / `heartbeat()` 接口
- **Ollama本地客户端** (`QwenOllamaClient`)：自动探测已安装模型，优先选择Qwen系列中文模型；支持思考/推理模型的流式输出兼容
- **DashScope云端客户端** (`QwenCloudClient`)：对接阿里云通义千问API（OpenAI兼容接口），支持流式SSE
- **Mock兜底客户端** (`MockQwenClient`)：无LLM环境下的演示模式
- **运行时热切换**：通过 `/api/llm/switch` 接口可在 Ollama → DashScope → Mock 之间实时切换，无需重启服务
- **心跳检测**：每个客户端实现 `heartbeat()` 返回模型状态与延迟

### 2. 对话管理器 (`src/dialogue/dialogue_manager.py`)

**亮点：多商户会话隔离 + RAG增强 + 人设注入 + 流式输出**

- **多会话管理**：按 `merchant_id` 隔离对话历史，每个商家维护独立的 `sessions`
- **RAG增强提示词**：自动拼接知识库检索结果到 system prompt，告知模型参考内容
- **Persona人设注入**：根据商家的AI形象配置（名字、性格描述），动态构建符合角色的 system prompt
- **流式SSE输出** (`stream_input`)：支持真实的逐token流式响应，提升用户体验
- **知识阈值配置**：可通过 `set_knowledge_threshold()` 调节RAG命中灵敏度

### 3. 知识库RAG系统 (`src/knowledge/knowledge_base.py`)

**亮点：TF-IDF+稳定规则双层检索 + 多商户隔离 + 数据持久化**

- **双层检索策略**：
  - 第一层「稳定规则命中」：针对"招牌推荐/营业时间/会员/配送"等高频场景，用关键词直接命中，保证演示稳定性
  - 第二层「TF-IDF + 余弦相似度」：char_wb级别的字符级n-gram向量化（适配中文），余弦相似度匹配
- **多商户数据隔离**：每个商家有独立的知识条目集合，`normalize_merchant_id()` 兜底保证不会串数据
- **JSON文件持久化**：`data/knowledge_base.json` 存储所有商家数据和知识库，支持运行时CRUD并即时保存
- **预置3个演示商家**：星巴克（咖啡）、墨香阁书店（图书）、云朵甜品店（甜品），各含4条知识

### 4. 意图识别系统 (`src/intent/`)

**亮点：关键词规则 + 否定词处理 + 可扩展 + ML训练管线**

- **关键词意图分类器** (`IntentClassifier`)：
  - 5类意图：greeting / thanks / goodbye / chitchat / faq
  - 置信度计算：基于关键词命中率 × 基础置信度 × 否定词惩罚
  - 动态管理：支持运行时 `add_intent()` / `remove_intent()` / `update_intent()`
- **ML训练管线** (`train.py`)：
  - 使用 SVM + TF-IDF 训练意图识别模型
  - 完整的 train/test split、模型评估、pickle持久化流程
  - 可通过 `npm run train` 一键训练

### 5. Web应用层 (`src/ui/web_ui.py`)

**这是整个项目最大的模块（约1478行），包含完整的Flask路由和业务逻辑**

**亮点：三角色权限体系 + 完整的商户生命周期 + AI能力全链路集成**

#### 5.1 权限与认证
- 三种角色：`customer`（顾客）、`merchant`（商户）、`super_admin`（平台管理员）
- `@login_required` 装饰器支持角色过滤
- 演示账户：user/123456, tea/123456, admin/123456

#### 5.2 客户端功能
- **首页** (`/`)：商家列表展示，分类浏览，搜索（按名称/类别/slogan），收藏
- **对话页** (`/chat`)：SSE流式AI对话，快捷回复，暗色模式，对话历史localStorage持久化，后端切换
- **个人中心**：修改资料、头像上传、收藏管理、对话历史
- **搜索页**：全文搜索 + 排序
- **全部分类**：按类别分组浏览

#### 5.3 商户端功能
- **工作台Dashboard** (`/admin`)：今日访客、今日咨询、累计收藏、店铺评分 四项指标
- **店铺信息编辑** (`/admin/store`)：名称、简介、分类、营业时间、地址、封面上传、头像上传
- **AI形象管理** (`/admin/persona`)：
  - 手动配置AI角色名称、性格描述
  - **LLM自动生成人设**：调用LLM根据店铺信息自动生成角色
  - **AI图片生成**：调用DashScope通义万相(wanx-v1)生成卡通风格客服头像，含轮询等待机制
  - **模板兜底生成**：LLM不可用时按品类（咖啡/茶饮/甜品/餐饮/生活服务）自动匹配模板
- **知识库管理** (`/admin/dashboard`)：FAQ的CRUD、模拟咨询测试
- **智能导入**：粘贴原始店铺资料文本 → LLM自动提取生成4-8条FAQ问答对
- **消息中心** (`/admin/messages`)：查看用户咨询记录、筛选/搜索
- **设置** (`/admin/settings`)：修改密码、更换手机号、注销商铺

#### 5.4 平台管理端
- **入驻审核** (`/platform/admin`)：approve/reject商户申请，查看资质详情
- **商户管理**：查看/编辑/删除所有商户
- **平台统计**：用户数、商户数、待审核数、消息数、分类分布等

#### 5.5 官方客服系统
- 平台内置官方客服聊天（SSE流式），帮助用户了解平台功能
- LLM优先 + 关键词兜底回复策略
- 消息记录与管理

#### 5.6 其他功能
- 评论评分系统
- 商户页面访问/咨询统计追踪
- 对话清空接口

## 四、项目亮点总结

| 编号 | 亮点 | 说明 |
|------|------|------|
| 1 | **多后端LLM抽象层** | 基于ABC抽象基类设计，统一Ollama/DashScope/OpenAI/Mock四种后端，支持运行时热切换和心跳检测 |
| 2 | **RAG增强检索** | TF-IDF字符级向量化 + 稳定规则命中的双层检索策略，兼顾检索效果和演示稳定性 |
| 3 | **多商户多租户架构** | 商家数据、知识库、对话历史完全隔离，支持动态扩展商户 |
| 4 | **AI人设自动生成** | LLM根据店铺信息自动生成AI客服的角色名、性格描述和emoji头像，还可调用AI绘图生成头像图片 |
| 5 | **SSE流式对话** | 全链路支持Server-Sent Events流式输出，包括商家对话和官方客服，提升真实感和交互体验 |
| 6 | **完整的平台生态** | 覆盖顾客端、商户端、平台管理端三角色，形成完整的B2B2C闭环 |
| 7 | **智能知识导入** | 粘贴原始文本 → LLM自动提取生成结构化FAQ问答对，大幅降低商户入驻门槛 |
| 8 | **演示友好设计** | Mock兜底模式（无需GPU/LLM也能运行演示）、预置演示账号和商家数据、关键词兜底回复 |
| 9 | **暗色模式** | 全站支持亮色/暗色主题切换，跟随系统偏好，localStorage持久化 |
| 10 | **模块化架构** | 清晰的分层设计（LLM层 → 对话层 → 知识层 → Web层），每层职责单一，便于测试和扩展 |
| 11 | **意图识别系统** | 关键词规则 + SVM/ML训练双模式，否定词智能降权，可动态扩展意图类别 |
| 12 | **商户入驻审核流** | 完整的申请→审核→通过→自动创建商户的自动化流程 |

## 五、技术特色

- **Python ABC抽象基类**：LLM客户端层的标准设计模式
- **Flask路由组织**：装饰器鉴权 + 角色过滤，路由组织清晰
- **SSE (Server-Sent Events)**：使用 `stream_with_context` + generator实现真正的流式推送
- **TF-IDF + Cosine Similarity**：中文友好的char_wb级别的n-gram向量化
- **并发安全**：流式对话使用 `ThreadPoolExecutor` + 60s超时控制
- **数据持久化**：JSON文件存储，支持读写分离和增量更新
